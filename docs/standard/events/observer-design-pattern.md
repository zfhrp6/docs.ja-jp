# <a name="observer-design-pattern"></a><span data-ttu-id="ad230-102">オブサーバー デザイン パターン</span><span class="sxs-lookup"><span data-stu-id="ad230-102">Observer Design Pattern</span></span>
<span data-ttu-id="ad230-103">オブザーバー デザイン パターンでは、プロバイダーにサブスクライバーを登録して通知を受信することができます。</span><span class="sxs-lookup"><span data-stu-id="ad230-103">The observer design pattern enables a subscriber to register with and receive notifications from a provider.</span></span> <span data-ttu-id="ad230-104">このデザイン パターンは、プッシュ ベースの通知を必要とするあらゆるシナリオに適しています。</span><span class="sxs-lookup"><span data-stu-id="ad230-104">It is suitable for any scenario that requires push-based notification.</span></span> <span data-ttu-id="ad230-105">このパターンでは、1 つの*プロバイダー* (*サブジェクト*または*観察可能なオブジェクト*とも呼ばれます) と、0 個以上の*オブザーバー*を定義します。</span><span class="sxs-lookup"><span data-stu-id="ad230-105">The pattern defines a *provider* (also known as a *subject* or an *observable*) and zero, one, or more *observers*.</span></span> <span data-ttu-id="ad230-106">プロバイダーにオブザーバーを登録すると、あらかじめ定義した条件、イベント、または状態変化が発生するたびに、プロバイダーのいずれかのメソッドが呼び出されて自動的にすべてのオブザーバーに通知されます。</span><span class="sxs-lookup"><span provider can also provide current state information to observers.</span></span> <span data-ttu-id="ad230-108">.NET Framework でオブザーバー デザイン パターンを適用するには、ジェネリック インターフェイスの ## <a name="applying-the-pattern"></a><span data-ttu-id="ad230-110">パターンの適用</span><span class="sxs-lookup"><span data-stu-id="ad230-110">Applying the Pattern</span></span>  
 <span data-ttu-id="ad230-111">オブザーバー デザイン パターンは、データ ソース (ビジネス ロジック) 層とユーザー インターフェイス (表示) 層のように、2 つの異なるコンポーネントまたはアプリケーション層を明確に分離できるため、分散型のプッシュ ベースの通知に適しています。</span><span class="sxs-lookup"><span data-stu-id="ad230-111">The observer design pattern is suitable for distributed push-based notifications, because it supports a clean separation between two different components or application layers, such as a data source (business logic) layer and a user interface (display) layer.</span></span> <span data-ttu-id="ad230-112">このパターンは、プロバイダーがコールバックを使用して現在の情報をクライアントに提供するあらゆる場面で実装できます。</span><span class="sxs-lookup"><span data-stu-id="ad230-112">The pattern can be implemented whenever a provider uses callbacks to supply its clients with current information.</span></span>  
  
 <span data-ttu-id="ad230-113">このパターンを実装するために必要なものを以下に示します。</span><span class="sxs-lookup"><span data-stu-id="ad230-113">Implementing the pattern requires that you provide the following:</span></span>  
  
-   <span data-ttu-id="ad230-114">プロバイダー (サブジェクト)。オブザーバーに通知を送信するオブジェクトです。</span><span class="sxs-lookup"><span data-stu-id="ad230-114">A provider or subject, which is the object that sends notifications to observers.</span></span> <span data-ttu-id="ad230-115">プロバイダーは、<xref:System.IObservable%601> インターフェイスを実装するクラスまたは構造体で、</span><span class="sxs-lookup"><span data-stu-id="ad230-115">A provider is a class or structure that implements the <xref:System.IObservable%601> interface.</span></span> <span data-ttu-id="ad230-116"><xref:System.IObservable%601.Subscribe%2A?displayProperty=nameWithType> という 1 つのメソッドを実装する必要があります。このメソッドは、プロバイダーからの通知を受信するオブザーバーによって呼び出されます。</span><span class="sxs-lookup"><span data-stu-id="ad230-116">The provider must implement a single method, <xref:System.IObservable%601.Subscribe%2A?displayProperty=nameWithType>, which is called by observers that wish to receive notifications from the provider.</span></span>  
  
-   <span data-ttu-id="ad230-117">オブザーバー。プロバイダーからの通知を受信するオブジェクトです。</span><span class="sxs-lookup"><span data-stu-id="ad230-117">An observer, which is an object that receives notifications from a provider.</span></span> <span data-ttu-id="ad230-118">オブザーバーは、<xref:System.IObserver%601> インターフェイスを実装するクラスまたは構造体で、</span><span class="sxs-lookup"><span data-stu-id="ad230-118">An observer is a class or structure that implements the <xref:System.IObserver%601> interface.</span></span> <span data-ttu-id="ad230-119">オブザーバーでは、次の 3 つのメソッドを実装する必要があります。これらはすべてプロバイダーによって呼び出されます。</span><span class="sxs-lookup"><span data-stu-id="ad230-119">The observer must implement three methods, all of which are called by the provider:</span></span>  
  
    -   <span data-ttu-id="ad230-120"><xref:System.IObserver%601.OnNext%2A?displayProperty=nameWithType>。新しい情報または現在の情報をオブザーバーに提供します。</span><span class="sxs-lookup"><span data-stu-id="ad230-120"><xref:System.IObserver%601.OnNext%2A?displayProperty=nameWithType>, which supplies the observer with new or current information.</span></span>  
  
    -   <span data-ttu-id="ad230-121"><xref:System.IObserver%601.OnError%2A?displayProperty=nameWithType>。エラーが発生したことをオブザーバーに通知します。</span><span class="sxs-lookup"><span data-stu-id="ad230-121"><xref:System.IObserver%601.OnError%2A?displayProperty=nameWithType>, which informs the observer that an error has occurred.</span></span>  
  
    -   <span data-ttu-id="ad230-122"><xref:System.IObserver%601.OnCompleted%2A?displayProperty=nameWithType>。プロバイダーが通知の送信を完了したことを示します。</span><span class="sxs-lookup"><span data-stu-id="ad230-122"><xref:System.IObserver%601.OnCompleted%2A?displayProperty=nameWithType>, which indicates that the provider has finished sending notifications.</span></span>  
  
-   <span data-ttu-id="ad230-123">プロバイダーがオブザーバーを追跡するための機構。</span><span class="sxs-lookup"><span data-stu-id="ad230-123">A mechanism that allows the provider to keep track of observers.</span></span> <span data-ttu-id="ad230-124">通常、プロバイダーでは、<xref:System.Collections.Generic.List%601?displayProperty=nameWithType> オブジェクトなどのコンテナー オブジェクトを使用して、通知にサブスクライブした <xref:System.IObserver%601> の実装への参照を保持します。</span><span class="sxs-lookup"><span data-stu-id="ad230-124">Typically, the provider uses a container object, such as a <xref:System.Collections.Generic.List%601?displayProperty=nameWithType> object, to hold references to the <xref:System.IObserver%601> implementations that have subscribed to notifications.</span></span> <span data-ttu-id="ad230-125">このようにストレージ コンテナーを使用すると、プロバイダーで必要に応じていくつでもオブザーバーを処理できます。</span><span class="sxs-lookup"><span data-stu-id="ad230-125">Using a storage container for this purpose enables the provider to handle zero to an unlimited number of observers.</span></span> <span data-ttu-id="ad230-126">オブザーバーが通知を受信する順序は定義されていないため、プロバイダーで自由に順序を決定できます。</span><span class="sxs-lookup"><span data-stu-id="ad230-126">The order in which observers receive notifications is not defined; the provider is free to use any method to determine the order.</span></span>  
  
-   <span data-ttu-id="ad230-127">通知が完了したときにプロバイダーがオブザーバーを削除できるようにするための <xref:System.IDisposable> の実装。</span><span class="sxs-lookup"><span data-stu-id="ad230-127">An <xref:System.IDisposable> implementation that enables the provider to remove observers when notification is complete.</span></span> <span data-ttu-id="ad230-128">オブザーバーは、<xref:System.IDisposable> の実装への参照を <xref:System.IObservable%601.Subscribe%2A> メソッドから受け取ります。このため、プロバイダーが通知の送信を終了する前に、オブザーバーで <xref:System.IDisposable.Dispose%2A?displayProperty=nameWithType> メソッドを呼び出してサブスクリプションを解除することもできます。</span><span class="sxs-lookup"><span data-stu-id="ad230-128">Observers receive a reference to the <xref:System.IDisposable> implementation from the <xref:System.IObservable%601.Subscribe%2A> method, so they can also call the <xref:System.IDisposable.Dispose%2A?displayProperty=nameWithType> method to unsubscribe before the provider has finished sending notifications.</span></span>  
  
-   <span data-ttu-id="ad230-129">プロバイダーがオブザーバーに送信するデータを含むオブジェクト。</span><span class="sxs-lookup"><span data-stu-id="ad230-129">An object that contains the data that the provider sends to its observers.</span></span> <span data-ttu-id="ad230-130">このオブジェクトの型は、<xref:System.IObservable%601> インターフェイスと <xref:System.IObserver%601> インターフェイスのジェネリック型パラメーターに対応します。</span><span class="sxs-lookup"><span data-stu-id="ad230-130">The type of this object corresponds to the generic type parameter of the <xref:System.IObservable%601> and <xref:System.IObserver%601> interfaces.</span></span> <span data-ttu-id="ad230-131">このオブジェクトは、<xref:System.IObservable%601> の実装と同じにすることもできますが、別の型にするのが最も一般的です。</span><span class="sxs-lookup"><span data-stu-id="ad230-131">Although this object can be the same as the <xref:System.IObservable%601> implementation, most commonly it is a separate type.</span></span>  
  
> [!NOTE]
>  <span data-ttu-id="ad230-132">オブザーバー デザイン パターンを実装するだけでなく、<xref:System.IObservable%601> インターフェイスと <xref:System.IObserver%601> インターフェイスを使用して構築されたライブラリを使用することもできます。</span><span class="sxs-lookup"><span data-stu-id="ad230-132">In addition to implementing the observer design pattern, you may be interested in exploring libraries that are built using the <xref:System.IObservable%601> and <xref:System.IObserver%601> interfaces.</span></span> <span data-ttu-id="ad230-133">たとえば、[.NET (Rx) の反応拡張](https://msdn.microsoft.com/library/hh242985.aspx)は、非同期プログラミングをサポートする一連の拡張メソッドと LINQ 標準シーケンス演算子で構成します。</span><span class="sxs-lookup"><span data-stu-id="ad230-133">For example, [Reactive Extensions for .NET (Rx)](https://msdn.microsoft.com/library/hh242985.aspx) consist of a set of extension methods and LINQ standard sequence operators to support asynchronous programming.</span></span>
