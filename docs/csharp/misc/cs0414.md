# <a name="compiler-warning-level-3-cs0414"></a><span data-ttu-id="3dade-102">コンパイラの警告 (レベル 3) CS0414</span><span class="sxs-lookup"><span data-stu-id="3dade-102">Compiler Warning (level 3) CS0414</span></span>
<span data-ttu-id="3dade-103">private フィールド 'field' は割り当てられていますが、その値が使用されていません</span><span class="sxs-lookup"><span data-stu-id="3dade-103">The private field 'field' is assigned but its value is never used</span></span>  
  
 <span data-ttu-id="3dade-104">この警告は、コンパイラで、変数が参照されていないことを確認できるいくつかのシナリオで発生する可能性があります。</span><span class="sxs-lookup"><span data-stu-id="3dade-104">This warning can occur in several scenarios in which the compiler can verify that a variable is never referenced:</span></span>  
  
-   <span data-ttu-id="3dade-105">private フィールドに定数値が割り当てられていますが、これ以降に一切読み取られていません。</span><span class="sxs-lookup"><span data-stu-id="3dade-105">A private field is assigned a constant value but is never subsequently read.</span></span> <span data-ttu-id="3dade-106">不要な割り当ては、パフォーマンスに影響する可能性があります。</span><span class="sxs-lookup"><span data-stu-id="3dade-106">The unnecessary assignment could effect performance.</span></span> <span data-ttu-id="3dade-107">フィールドの削除を検討してください。</span><span class="sxs-lookup"><span data-stu-id="3dade-107">Consider removing the field.</span></span>  
  
-   <span data-ttu-id="3dade-108">private または内部の静的フィールドの初期化子にのみ、定数値が割り当てられています。</span><span class="sxs-lookup"><span data-stu-id="3dade-108">A private or internal static field is assigned a constant value only in the initializer.</span></span> <span data-ttu-id="3dade-109">フィールドを const に変更することを検討してください。</span><span class="sxs-lookup"><span data-stu-id="3dade-109">Consider changing the field to a const.</span></span>  
  
-   <span data-ttu-id="3dade-110">private または内部フィールドに定数値が割り当てられており、#ifdef ディレクティブによって除外されたブロック内でのみ使用されています。</span><span class="sxs-lookup"><span data-stu-id="3dade-110">A private or internal field is assigned constant values and only used in blocks that are excluded by #ifdef directives.</span></span> <span data-ttu-id="3dade-111">#ifdef ブロック内にフィールドを配置することを検討してください。</span><span class="sxs-lookup"><span data-stu-id="3dade-111">Consider putting the field inside the #ifdef block.</span></span>  
  
-   <span data-ttu-id="3dade-112">private または内部フィールドの複数の場所に定数値が割り当てられていますが、それ以外はアクセスされていません。</span><span class="sxs-lookup"><span data-stu-id="3dade-112">A private or internal field is assigned constant values in multiple locations but not otherwise accessed.</span></span> <span data-ttu-id="3dade-113">フィールドが不要な場合は、削除することを検討してください。</span><span class="sxs-lookup"><span data-stu-id="3dade-113">If you do not need the field, consider removing it.</span></span> <span data-ttu-id="3dade-114">削除しない場合は、何らかの適切な方法でこのフィールドを使用します。</span><span class="sxs-lookup"><span data-stu-id="3dade-114">Otherwise, use it in some appropriate way.</span></span>  
  
 <span data-ttu-id="3dade-115">その他の場合、または提案されている回避策が受け入れ不可の場合は、#pragma 0414 を使用します。</span><span class="sxs-lookup"><span data-stu-id="3dade-115">In other situations, or where the suggested workaround is not acceptable, use #pragma 0414.</span></span>  
  
 <span data-ttu-id="3dade-116">次の例では、CS0414 が生成される仕組みの 1 つを示しています。</span><span class="sxs-lookup"><span data-stu-id="3dade-116">The following sample shows one way in which CS0414 will be generated:</span></span>  
  
```  
// CS0414  
// compile with: /W3  
class C  
{  
   private int i = 1;  // CS0414  
  
   public static void Main()  
   { }  
}  
```  
  
 <span data-ttu-id="3dade-117">**メモ** 変数 `i` が `protected or public`として宣言されている場合、エラーは生成されません。これは、派生クラスでこの変数が使用されるのか、その他のクライアント コードによってクラスがインスタンス化され、変数が参照されるのかどうかが、コンパイラで認識できないためです。</span><span class="sxs-lookup"><span data-stu-id="3dade-117">**Note** If the variable `i` is declared as `protected or public`, no error will be generated because the compiler cannot know whether a derived class might use it or some other client code might instantiate the class and reference the variable</span></span>
