# <a name="compiler-error-cs1921"></a><span data-ttu-id="5c931-102">コンパイラ エラー CS1921</span><span class="sxs-lookup"><span data-stu-id="5c931-102">Compiler Error CS1921</span></span>
<span data-ttu-id="5c931-103">'method' に最も適しているオーバーロード メソッドには、初期化子要素の正しくないシグネチャが含まれます。</span><span class="sxs-lookup"><span data-stu-id="5c931-103">The best overloaded method match for 'method' has wrong signature for the initializer element.</span></span> <span data-ttu-id="5c931-104">初期化可能な Add は、アクセス可能なインスタンス メソッドでなければなりません。</span><span class="sxs-lookup"><span data-stu-id="5c931-104">The initializable Add must be an accessible instance method.</span></span>  
  
 <span data-ttu-id="5c931-105">このエラーは、パブリックでない非静的 `Add` メソッドを持つクラスでコレクション初期化子を使用しようとしたときに生成されます。</span><span class="sxs-lookup"><span data-stu-id="5c931-105">This error is generated when you try to use a collection initializer with a class that has no public non-static `Add` method.</span></span> <span data-ttu-id="5c931-106">`Add` メソッドの保護レベル (`private`、`protected`、`internal`) が原因でメソッドがアクセスできない場合は、CS0122 が発生するため、このエラーが発生した場合は、メソッドが `static` として定義されていることを示していると考えられます。</span><span class="sxs-lookup"><span data-stu-id="5c931-106">If the `Add` method is not accessible because of its protection level (`private`, `protected`, `internal`) then you will get CS0122, so that this error probably means that the method is defined as `static`.</span></span>  
  
## <a name="example"></a><span data-ttu-id="5c931-107">例</span><span class="sxs-lookup"><span data-stu-id="5c931-107">Example</span></span>  
 <span data-ttu-id="5c931-108">次の例では CS1921 が生成されます。</span><span class="sxs-lookup"><span data-stu-id="5c931-108">The following example generates CS1921:</span></span>  
  
```csharp  
// cs1921.cs  
using System.Collections;  
public class C : CollectionBase  
{  
    public static void Add(int i)  
    {  
    }  
}

    public static void Main()  
    {  
        var collection = new C { 1, 2, 3 }; // CS1921  
    }  
}  
```
