# <a name="compiler-warning-level-2-cs0467"></a><span data-ttu-id="812cf-102">コンパイラの警告 (レベル 2) CS0467</span><span class="sxs-lookup"><span data-stu-id="812cf-102">Compiler Warning (level 2) CS0467</span></span>
<span data-ttu-id="812cf-103">メソッド '<メソッド>' とメソッド以外の 'non-method' があいまいです。メソッド グループを使用しています。</span><span class="sxs-lookup"><span data-stu-id="812cf-103">Ambiguity between method 'method' and non-method 'non-method'.Using method group.</span></span>  
  
 <span data-ttu-id="812cf-104">別のインターフェイスからメンバーが継承され、同じシグネチャによってあいまいさのエラーが発生します。</span><span class="sxs-lookup"><span data-stu-id="812cf-104">Inherited members from different interfaces that have the same signature  cause an ambiguity error.</span></span>  
  
## <a name="example"></a><span data-ttu-id="812cf-105">例</span><span class="sxs-lookup"><span data-stu-id="812cf-105">Example</span></span>  
 <span data-ttu-id="812cf-106">次の例では CS0467 が生成されます。</span><span class="sxs-lookup"><span data-stu-id="812cf-106">The following example generates CS0467.</span></span>  
  
```csharp  
// CS0467.cs  
interface IList   
{  
    int Count { get; set; }  
}  
  
interface ICounter  
{  
    void Count(int i);  
}  
  
interface IListCounter : IList, ICounter {}  
  
class Driver   
{  
    void Test(IListCounter x)  
    {  
        // The following line causes the warning. The assignment also  
        // causes an error because you can't assign a value to a method.  
        x.Count = 1;  
        x.Count(3);     
        // To resolve the warning, you can change the name of the method or   
        // the property.  
  
        // You also can disambiguate by specifying IList or ICounter.  
        ((IList)x).Count = 1;  
        ((ICounter)x).Count(3);  
    }  
  
    static void Main()   
    {  
    }  
}  
```
