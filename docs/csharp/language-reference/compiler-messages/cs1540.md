# <a name="compiler-error-cs1540"></a>コンパイラ エラー CS1540
'<型 1>' 型の修飾子をとおしてプロテクト メンバー '<メンバー>' にアクセスすることはできません。修飾子は '<型 2>' 型、またはそれから派生したものでなければなりません。  
  
 派生[クラス](../../../csharp/language-reference/keywords/class.md)は、基本クラスのインスタンスを通じて基本クラスのプロテクト メンバーにアクセスすることはできません。 派生クラス内で宣言された基本クラスのインスタンスが、実行時に、派生元が同じであること以外には派生クラスとの関連性のない別の型のインスタンスになることがあります。 プロテクト メンバーにアクセスできるのは派生した型のみであるため、実行時に有効でない可能性があるプロテクト メンバーにアクセスする試みは、コンパイラによって無効とマークされます。  
  
 次の例の `Employee` クラスでは、`emp2` と `emp3` の両方がコンパイル時に `Person` 型になりますが、`emp2` は実行時に `Manager` 型になります。 `Employee` は `Manager` から派生しないため、基本クラスである `Person` のプロテクト メンバーに、`Manager` クラスのインスタンスを通じてアクセスすることはできません。 コンパイラは、2 つの `Person` オブジェクトの実行時の型が何になるかを判断できません。 したがって、`emp2` からの呼び出しと `emp3` からの呼び出しによってコンパイラ エラー CS1540 が発生します。  
  
```csharp  
using System;  
using System.Text;  
  
namespace CS1540  
{  
    class Program1  
    {  
        static void Main()  
        {  
            Employee.PreparePayroll();  
        }  
    }  
  
    class Person  
    {  
        protected virtual void CalculatePay()   
        {  
            Console.WriteLine("CalculatePay in Person class.");  
        }  
    }  
  
    class Manager : Person  
    {  
        protected override void CalculatePay()   
        {  
            Console.WriteLine("CalculatePay in Manager class.");   
  
        }  
    }  
  
    class Employee : Person  
    {  
        public static void PreparePayroll()  
        {  
            Employee emp1 = new Employee();  
            Person emp2 = new Manager();  
            Person emp3 = new Employee();  
            // The following line calls the method in the Employee base class,  
            // Person.  
            emp1.CalculatePay();   
  
            // The following lines cause compiler error CS1540. The compiler   
            // cannot determine at compile time what the run-time types of   
            // emp2 and emp3 will be.  
            //emp2.CalculatePay();   
            //emp3.CalculatePay();  
  
        }  
    }  
}  
```
