# <a name="compiler-error-cs0269"></a><span data-ttu-id="95a91-102">コンパイラ エラー CS0269</span><span class="sxs-lookup"><span data-stu-id="95a91-102">Compiler Error CS0269</span></span>
<span data-ttu-id="95a91-103">未割り当ての out パラメーター 'parameter' が使用されました</span><span class="sxs-lookup"><span data-stu-id="95a91-103">Use of unassigned out parameter 'parameter'</span></span>  
  
 <span data-ttu-id="95a91-104">コンパイラは、使用前に out パラメーターに値が割り当てられたことを確認できなかった可能性があります。その値は、割り当てられたときに未定義だった可能性があります。</span><span class="sxs-lookup"><span data-stu-id="95a91-104">The compiler could not verify that the out parameter was assigned a value before it was used; its value may be undefined when assigned.</span></span> <span data-ttu-id="95a91-105">値にアクセスする前に、呼び出されるメソッドの `out` パラメーターに必ず値を割り当てます。</span><span class="sxs-lookup"><span data-stu-id="95a91-105">Be sure to assign a value to `out` parameters in the called method before accessing the value.</span></span> <span data-ttu-id="95a91-106">渡された変数の値を使用する必要がある場合、代わりに `ref` パラメーターを使用します。</span><span class="sxs-lookup"><span data-stu-id="95a91-106">If you need to use the value of the variable passed in, use a `ref` parameter instead.</span></span> <span data-ttu-id="95a91-107">詳細については、「[パラメーターの引き渡し](../../../csharp/programming-guide/classes-and-structs/passing-parameters.md)」を参照してください。</span><span class="sxs-lookup"><span data-stu-id="95a91-107">For more information, see [Passing Parameters](../../../csharp/programming-guide/classes-and-structs/passing-parameters.md).</span></span>  
  
## <a name="example"></a><span data-ttu-id="95a91-108">例</span><span class="sxs-lookup"><span data-stu-id="95a91-108">Example</span></span>  
 <span data-ttu-id="95a91-109">次の例では CS0269 が生成されます。</span><span class="sxs-lookup"><span data-stu-id="95a91-109">The following sample generates CS0269:</span></span>  
  
```csharp  
// CS0269.cs  
class C  
{  
    public static void F(out int i)  
    // One way to resolve the error is to use a ref parameter instead  
    // of an out parameter.  
    //public static void F(ref int i)  
    {  
        // The following line causes a compiler error because no value  
        // has been assigned to i.  
        int k = i;  // CS0269  
        i = 1;  
        // The error does not occur if the order of the two previous   
        // lines is reversed.  
    }  
  
    public static void Main()  
    {  
        int myInt = 1;  
        F(out myInt);  
        // If the declaration of method F is changed to require a ref  
        // parameter, ref must be specified in the call as well.  
        //F(ref myInt);  
    }  
}  
```  
  
## <a name="example"></a><span data-ttu-id="95a91-110">例</span><span class="sxs-lookup"><span data-stu-id="95a91-110">Example</span></span>  
 <span data-ttu-id="95a91-111">これは、変数の初期化が try ブロックで発生する場合にも発生することがあります。コンパイラは成功を確認できません。</span><span class="sxs-lookup"><span data-stu-id="95a91-111">This could also occur if the initialization of a variable occurs in a try block, which the compiler is unable to verify will execute successfully:</span></span>  
  
```csharp  
// CS0269b.cs  
class C  
{  
    public static void F(out int i)  
    {  
        try  
        {  
            // Assignment occurs, but compiler can't verify it  
            i = 1;  
        }  
        catch  
        {  
        }  
  
        int k = i;  // CS0269  
        i = 1;  
    }  
  
    public static void Main()  
    {  
        int myInt;  
        F(out myInt);  
    }  
}  
```
