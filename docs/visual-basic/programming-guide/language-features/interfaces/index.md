# <a name="interfaces-visual-basic"></a><span data-ttu-id="0a271-102">インターフェイス (Visual Basic)</span><span class="sxs-lookup"><span data-stu-id="0a271-102">Interfaces (Visual Basic)</span></span>
<span data-ttu-id="0a271-103">*インターフェイス*は、クラスが実装できるプロパティ、メソッド、およびイベントを定義します。</span><span class="sxs-lookup"><span data-stu-id="0a271-103">*Interfaces* define the properties, methods, and events that classes can implement.</span></span> <span data-ttu-id="0a271-104">インターフェイスでは、密接に関連するプロパティ、メソッド、およびイベントの小さなグループとして機能を定義できます。これにより、既存のコードを損なうことなく、インターフェイスを拡張して実装を開発できるため、互換性の問題を減らすことができます。</span><span class="sxs-lookup"><span data-stu-id="0a271-104">Interfaces allow you to define features as small groups of closely related properties, methods, and events; this reduces compatibility problems because you can develop enhanced implementations for your interfaces without jeopardizing existing code.</span></span> <span data-ttu-id="0a271-105">追加のインターフェイスと実装を開発することで、いつでも新しい機能を追加できます。</span><span class="sxs-lookup"><span data-stu-id="0a271-105">You can add new features at any time by developing additional interfaces and implementations.</span></span>  
  
 <span data-ttu-id="0a271-106">クラスの継承の代わりにインターフェイスを使用する方が望ましい理由が、その他にもいくつかあります。</span><span class="sxs-lookup"><span data-stu-id="0a271-106">There are several other reasons why you might want to use interfaces instead of class inheritance:</span></span>  
  
-   <span data-ttu-id="0a271-107">インターフェイスは、アプリケーションが特定の機能を提供するために関連性の低い多数のオブジェクトの種類を必要とする状況に、より適しています。</span><span class="sxs-lookup"><span data-stu-id="0a271-107">Interfaces are better suited to situations in which your applications require many possibly unrelated object types to provide certain functionality.</span></span>  
  
-   <span data-ttu-id="0a271-108">インターフェイスは、複数のインターフェイスを実装できる単一の実装を定義できるため、基底クラスよりも柔軟です。</span><span class="sxs-lookup"><span data-stu-id="0a271-108">Interfaces are more flexible than base classes because you can define a single implementation that can implement multiple interfaces.</span></span>  
  
-   <span data-ttu-id="0a271-109">インターフェイスは、基底クラスから実装を継承する必要がない状況に、より適しています。</span><span class="sxs-lookup"><span data-stu-id="0a271-109">Interfaces are better in situations in which you do not have to inherit implementation from a base class.</span></span>  
  
-   <span data-ttu-id="0a271-110">インターフェイスは、クラスの継承を使用できない場合に便利です。</span><span class="sxs-lookup"><span data-stu-id="0a271-110">Interfaces are useful when you cannot use class inheritance.</span></span> <span data-ttu-id="0a271-111">たとえば、構造体はクラスから継承できませんが、インターフェイスを実装できます。</span><span class="sxs-lookup"><span data-stu-id="0a271-111">For example, structures cannot inherit from classes, but they can implement interfaces.</span></span>  
  
## <a name="declaring-interfaces"></a><span data-ttu-id="0a271-112">インターフェイスの宣言</span><span class="sxs-lookup"><span data-stu-id="0a271-112">Declaring Interfaces</span></span>  
 <span data-ttu-id="0a271-113">インターフェイスの定義は、`Interface` ステートメントと `End Interface` ステートメントで囲みます。</span><span class="sxs-lookup"><span data-stu-id="0a271-113">Interface definitions are enclosed within the `Interface` and `End Interface` statements.</span></span> <span data-ttu-id="0a271-114">`Interface` ステートメントの後に、オプションで`Inherits` ステートメントを追加して、継承されるインターフェイスを 1 つ以上指定することができます。</span><span class="sxs-lookup"><span data-stu-id="0a271-114">Following the `Interface` statement, you can add an optional `Inherits` statement that lists one or more inherited interfaces.</span></span> <span data-ttu-id="0a271-115">`Inherits` ステートメントは、宣言内のコメントを除く他のすべてのステートメントより前に記述する必要があります。</span><span class="sxs-lookup"><span data-stu-id="0a271-115">The `Inherits` statements must precede all other statements in the declaration except comments.</span></span> <span data-ttu-id="0a271-116">インターフェイス定義の残りのステートメントは、`Event`、`Sub`、`Function`、`Property`、`Interface`、`Class`、`Structure`、および `Enum` ステートメントです。</span><span class="sxs-lookup"><span data-stu-id="0a271-116">The remaining statements in the interface definition should be `Event`, `Sub`, `Function`, `Property`, `Interface`, `Class`, `Structure`, and `Enum` statements.</span></span> <span data-ttu-id="0a271-117">インターフェイスには、`End Sub` や `End Property` など、実装コードや実装コードに関連付けられているステートメントを含めることはできません。</span><span class="sxs-lookup"><span data-stu-id="0a271-117">Interfaces cannot contain any implementation code or statements associated with implementation code, such as `End Sub` or `End Property`.</span></span>
