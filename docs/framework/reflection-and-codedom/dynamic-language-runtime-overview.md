-   <span data-ttu-id="ff179-165">呼び出しサイトのキャッシュ。</span><span class="sxs-lookup"><span data-stu-id="ff179-165">Call site caching.</span></span> <span data-ttu-id="ff179-166">*動的呼び出しサイト*とは、動的オブジェクトに `a + b` や `a.b()` などの操作を実行するコード内の場所です。</span><span class="sxs-lookup"><span data-stu-id="ff179-166">A *dynamic call site* is a place in the code where you perform an operation like `a + b` or `a.b()` on dynamic objects.</span></span> <span data-ttu-id="ff179-167">DLR は、`a` と `b` の特性 (通常はこれらのオブジェクトの型) や操作に関する情報をキャッシュします。</span><span class="sxs-lookup"><span data-stu-id="ff179-167">The DLR caches the characteristics of `a` and `b` (usually the types of these objects) and information about the operation.</span></span> <span data-ttu-id="ff179-168">同様の操作が以前に実行されていた場合、DLR は、必要なすべての情報をキャッシュから取得して、高速なディスパッチを実現します。</span><span class="sxs-lookup"><span data-stu-id="ff179-168">If such an operation has been performed previously, the DLR retrieves all the necessary information from the cache for fast dispatch.</span></span>  
  
-   <span data-ttu-id="ff179-169">動的オブジェクトの相互運用性。</span><span class="sxs-lookup"><span data-stu-id="ff179-169">Dynamic object interoperability.</span></span> <span data-ttu-id="ff179-170">DLR には、動的オブジェクトと操作を表し、言語実装者や動的ライブラリの作成者が使用できるクラスとインターフェイスのセットが用意されています。</span><span class="sxs-lookup"><span data-stu-id="ff179-170">The DLR provides a set of classes and interfaces that represent dynamic objects and operations and can be used by language implementers and authors of dynamic libraries.</span></span> <span data-ttu-id="ff179-171">このようなクラスやインターフェイスには、<xref:System.Dynamic.IDynamicMetaObjectProvider><xref:System.Dynamic.DynamicMetaObject><xref:System.Dynamic.DynamicObject> および <xref:System.Dynamic.ExpandoObject> があります。</span><span class="sxs-lookup"><span data-stu-id="ff179-171">These classes and interfaces include <xref:System.Dynamic.IDynamicMetaObjectProvider>, <xref:System.Dynamic.DynamicMetaObject>, <xref:System.Dynamic.DynamicObject>, and <xref:System.Dynamic.ExpandoObject>.</span></span>  
  
 <span data-ttu-id="ff179-172">DLR では、呼び出しサイトのバインダーを使用して、.NET Framework だけでなく、Silverlight や COM などの他のインフラストラクチャやサービスとも通信します。</span><span class="sxs-lookup"><span data-stu-id="ff179-172">The DLR uses binders in call sites to communicate not only with the .NET Framework, but with other infrastructures and services, including Silverlight and COM.</span></span> <span data-ttu-id="ff179-173">バインダーは、言語のセマンティクスをカプセル化し、式ツリーを使用して呼び出しサイトの操作を実行する方法を指定します。</span><span class="sxs-lookup"><span data-stu-id="ff179-173">Binders encapsulate a language's semantics and specify how to perform operations in a call site by using expression trees.</span></span> <span data-ttu-id="ff179-174">これにより、DLR を使用する動的言語および静的に型指定された言語で、ライブラリを共有し、DLR がサポートするすべてのテクノロジを利用できるようになります。</span><span class="sxs-lookup"><span data-stu-id="ff179-174">This enables dynamic and statically typed languages that use the DLR to share libraries and gain access to all the technologies that the DLR supports.</span></span>
